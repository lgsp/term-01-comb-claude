Rappels de cours & notions fondamentales

La combinatoire étudie le dénombrement des configurations possibles dans des ensembles finis. Elle est fondamentale en probabilités, en informatique et en algèbre.

Principe multiplicatif

Si une expérience se déroule en \(k\) étapes, l'étape \(i\) ayant \(n_i\) issues possibles (indépendamment des étapes précédentes), alors le nombre total d'issues est \(n_1 \times n_2 \times \cdots \times n_k\).
Exemple

Un code PIN à 4 chiffres (0–9) possède \(10^4 = 10\,000\) valeurs possibles. Si les chiffres doivent être distincts : \(10 \times 9 \times 8 \times 7 = 5\,040\).

Factorielle

Pour tout entier naturel \(n \geq 1\) : \[n! = n \times (n-1) \times \cdots \times 2 \times 1\] Par convention : \(0! = 1\).

Listes ordonnées (k-uplets)

Une liste ordonnée de \(k\) éléments tirés dans un ensemble de \(n\) éléments avec répétition possible est appelée un \(k\)-uplet. Le nombre de tels \(k\)-uplets est \(n^k\).

Permutations & arrangements

Permutations

Une permutation d'un ensemble de \(n\) éléments est un arrangement ordonné de ces \(n\) éléments (sans répétition). Le nombre de permutations est \(P_n = n!\).
Exemple — podium

10 coureurs s'affrontent. Le nombre de podiums (1er, 2e, 3e) distincts est \(A_{10}^3 = 10 \times 9 \times 8 = 720\). Le nombre de classements complets est \(10! = 3\,628\,800\).

Arrangements sans répétition

Un arrangement de \(k\) éléments parmi \(n\) (sans répétition, ordre important) est noté \(A_n^k\) : \[A_n^k = \frac{n!}{(n-k)!} = n \times (n-1) \times \cdots \times (n-k+1)\] avec \(0 \leq k \leq n\).
Démonstration — Principe multiplicatif
On choisit séquentiellement : le 1er élément parmi \(n\) choix, le 2e parmi \(n-1\) restants, …, le \(k\)-ème parmi \(n-k+1\) restants. Par le principe multiplicatif : \[A_n^k = n \cdot (n-1) \cdots (n-k+1) = \frac{n!}{(n-k)!} \quad \blacksquare\]
SituationFormuleExemple
Liste ordonnée avec répétition (\(k\) parmi \(n\))\(n^k\)Codes PIN : \(10^4\)
Arrangements sans répétition\(A_n^k = \frac{n!}{(n-k)!}\)Podium 3/10 : \(\frac{10!}{7!}=720\)

Combinaisons — coefficients binomiaux

Définition

Une combinaison de \(k\) éléments parmi \(n\) (ordre indifférent, sans répétition) est notée \(\binom{n}{k}\) (lu « \(k\) parmi \(n\) ») : \[\binom{n}{k} = \frac{n!}{k!(n-k)!} = \frac{A_n^k}{k!}\] avec \(0 \leq k \leq n\), et \(\binom{n}{k} = 0\) si \(k > n\).
Lien entre arrangements et combinaisons
Un arrangement de \(k\) éléments parmi \(n\) peut se construire en deux étapes :
  1. Choisir les \(k\) éléments (sans tenir compte de l'ordre) : \(\binom{n}{k}\) façons.
  2. Les ordonner : \(k!\) façons.

Donc \(A_n^k = \binom{n}{k} \times k!\), d'où \(\binom{n}{k} = \frac{A_n^k}{k!} = \frac{n!}{k!(n-k)!}\). \(\blacksquare\)

Propriétés fondamentales

Propriétés à retenir

• Symétrie : \(\binom{n}{k} = \binom{n}{n-k}\)  (choisir \(k\) éléments ↔ rejeter \(n-k\) éléments)

• Cas extrêmes : \(\binom{n}{0} = \binom{n}{n} = 1\)

• Relation de Pascal : \(\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}\)

• Somme d'une ligne : \(\displaystyle\sum_{k=0}^{n}\binom{n}{k} = 2^n\)

Démonstration — Relation de Pascal
Soit \(E\) un ensemble à \(n\) éléments, et \(a \in E\) fixé. Les parties à \(k\) éléments de \(E\) se divisent en deux catégories :
  • Celles qui contiennent \(a\) : il reste à choisir \(k-1\) éléments parmi les \(n-1\) autres → \(\binom{n-1}{k-1}\) façons.
  • Celles qui ne contiennent pas \(a\) : on choisit \(k\) éléments parmi les \(n-1\) autres → \(\binom{n-1}{k}\) façons.

Ces deux cas sont disjoints et exhaustifs, donc \(\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}\). \(\blacksquare\)
Démonstration — Propriété de symétrie
\[\binom{n}{n-k} = \frac{n!}{(n-k)!\,k!} = \frac{n!}{k!(n-k)!} = \binom{n}{k} \quad \blacksquare\] Interprétation : choisir \(k\) éléments à inclure revient à choisir \(n-k\) éléments à exclure.

Triangle de Pascal

Le triangle de Pascal est une représentation visuelle des coefficients binomiaux, organisée en lignes. La ligne \(n\) contient \(\binom{n}{0}, \binom{n}{1}, \ldots, \binom{n}{n}\).

Chaque coefficient est la somme des deux coefficients qui le surplombent. La première et la dernière case de chaque ligne valent 1. Cette règle est exactement la relation de Pascal : \(\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}\).

Formule du binôme de Newton

Pour tout \(n \in \mathbb{N}\) et tous réels \(a, b\) : \[(a+b)^n = \sum_{k=0}^{n} \binom{n}{k} a^{n-k} b^k = \binom{n}{0}a^n + \binom{n}{1}a^{n-1}b + \cdots + \binom{n}{n}b^n\]
Démonstration par récurrence
Initialisation : Pour \(n=0\), \((a+b)^0 = 1 = \binom{0}{0}a^0b^0\). ✓

Hérédité : Supposons la formule vraie au rang \(n\). Alors : \[(a+b)^{n+1} = (a+b)(a+b)^n = (a+b)\sum_{k=0}^{n}\binom{n}{k}a^{n-k}b^k\] En développant et en regroupant les termes en \(a^{n+1-k}b^k\) : \[= \sum_{k=0}^{n}\binom{n}{k}a^{n+1-k}b^k + \sum_{k=0}^{n}\binom{n}{k}a^{n-k}b^{k+1}\] En réindexant la seconde somme (\(j = k+1\)) et en utilisant la relation de Pascal \(\binom{n}{k}+\binom{n}{k-1}=\binom{n+1}{k}\) : \[= \sum_{k=0}^{n+1}\binom{n+1}{k}a^{n+1-k}b^k \quad \blacksquare\]

Corollaires importants

Cas particuliers (à connaître)

• \(a=b=1\) : \(\displaystyle\sum_{k=0}^n \binom{n}{k} = 2^n\)

• \(a=1, b=-1\) : \(\displaystyle\sum_{k=0}^n (-1)^k\binom{n}{k} = 0\) pour \(n\geq 1\)

Démonstrations du programme

Le programme officiel exige la connaissance de plusieurs démonstrations exemplaires. Les voici présentées avec rigueur.

Démo 1 — Formule \(\binom{n}{k} = \frac{n!}{k!(n-k)!}\)

On compte les arrangements de \(k\) éléments parmi \(n\) de deux façons :
  1. Directement : \(A_n^k = n(n-1)\cdots(n-k+1) = \dfrac{n!}{(n-k)!}\).
  2. En décomposant : choisir les \(k\) éléments → \(\binom{n}{k}\) façons, puis les ordonner → \(k!\) façons. Donc \(A_n^k = \binom{n}{k} \cdot k!\).
En égalisant : \(\binom{n}{k} \cdot k! = \dfrac{n!}{(n-k)!}\), d'où \(\binom{n}{k} = \dfrac{n!}{k!(n-k)!}\). \(\blacksquare\)

Démo 2 — Relation de Pascal (démontrée ci-dessus)

\(\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}\) — démonstration combinatoire par partition d'un ensemble (voir section 03).

Démo 3 — Formule du binôme (démonstration par récurrence)

\((a+b)^n = \sum_{k=0}^{n}\binom{n}{k}a^{n-k}b^k\) — démonstration par récurrence (voir section 05).

Démo 4 — \(\sum_{k=0}^n \binom{n}{k} = 2^n\)

Méthode algébrique : Appliquer la formule du binôme avec \(a=b=1\) : \[2^n = (1+1)^n = \sum_{k=0}^{n}\binom{n}{k} 1^{n-k} \cdot 1^k = \sum_{k=0}^{n}\binom{n}{k} \quad \blacksquare\]
Méthode combinatoire : \(\displaystyle\sum_{k=0}^n\binom{n}{k}\) compte toutes les parties d'un ensemble à \(n\) éléments (on les regroupe par taille). Or \(\mathcal{P}(E)\) a \(2^n\) éléments (pour chaque élément : inclus ou non, \(2\) choix). \(\blacksquare\)
